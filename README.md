🚀 **Featured Project**

### **Project 1 — Real-Time Notification Service (Elixir + PostgreSQL)**

Built a highly scalable real-time notification infrastructure using Elixir, Phoenix, and PostgreSQL based on OTP design principles. Leveraged Phoenix Channels and PubSub for efficient WebSocket fan-out while PostgreSQL LISTEN/NOTIFY bridged database events directly into the Elixir process ecosystem.

Implemented GenServers for isolated per-user state management and Supervisor trees for automatic fault recovery and system resilience. The architecture heavily utilized Elixir’s actor model where every connected user operates as a lightweight independent process (~2KB memory footprint), enabling the system to sustain hundreds of thousands of concurrent connections on modest hardware.

Focused on:

* Fault-tolerant distributed architecture
* Real-time event-driven communication
* Concurrent process management
* WebSocket infrastructure at scale
* High availability and system resilience
* PostgreSQL event streaming and pub/sub patterns
